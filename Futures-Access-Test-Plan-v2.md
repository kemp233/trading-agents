## 背景与方向调整

根据东证期货《接入终端测试反馈表 V1.0.1》的要求，系统需在约 **2026年3月11日** 前完成接入测试准备。本方案将开发重心从"加密货币+期货并行"调整为 **优先聚焦期货板块**，加密货币板块待接入测试通过后另行设立。[^1]

接入测试由期货公司主导，依据中国期货业协会发布的《期货程序化交易系统功能测试指引》(T/ZQX 0004—2025)，涵盖接口适应性、异常监测、阈值管理、错误防范、应急处置、日志记录六大类共 12 项测试指标。[^2][^3]

### AppID 编制规范（备查）

AppID 必须符合中国期货市场监控中心格式要求：[^4][^5]

- 格式：`终端厂商名称_终端软件名称_版本号`
- 最大长度：厂商名10字符 + 软件名10字符 + 版本号8字符
- 个人开发者厂商名称固定为 `client`
- 同一终端软件在不同期货公司应保持一致

## 接入测试完整流程

根据东证期货及行业通行做法，流程如下：[^6][^7]

1. **联系客户经理** → 提出穿透式接入测试申请
2. **填表提交 AppID** → 提供软件信息、硬件信息（CPU序列号、MAC地址、IP等）
3. **拿到 AuthCode** → 期货公司审核后下发测试账号 + 授权编码（通常1个工作日）
4. **CTP 仿真测试** → 使用测试环境完成全部 12 项指标的功能验证
5. **期货公司校验** → 东证技术人员见证确认，填写反馈表
6. **实盘接入** → 校验通过后，AppID 和 AuthCode 添加到实盘 CTP

## 系统说明书大纲

系统说明书需涵盖以下内容，供期货公司审核：[^8][^2]

### 1. 基本信息
- 软件名称：`aiagentts`，版本号：`1.0.0`，AppID：`client_aiagentts_1.0.0`
- 开发主体：个人，开发人：方馒涵
- 适用柜台：CTP

### 2. 系统功能概述
- 程序化交易系统整体架构（Multi-Agent 多智能体协作）
- 交易指令执行方式（Outbox Pattern 原子下单）
- 支持品种（中国期货市场主力合约）

### 3. 风控功能说明
- **异常监测**: 连接状态监测、报撤单统计、重复报单检测
- **阈值管理**: 可配置指标阈值 + 多级预警
- **交易指令检查**: 合约有效性、最小变动价位、单笔最大手数、涨跌停价格
- **错误防范**: 柜台错误码展示 + 自动拦截
- **应急处置**: 手动/自动暂停交易、批量撤单、分阶段平仓
- **日志记录**: 交易日志、运行记录、监测数据、错误提示

### 4. 技术架构
- Python 3.11 + asyncio
- SQLite WAL 模式持久化
- vn.py CTP API 接口（`vnpy_ctp`）
- Streamlit 管理界面

### 5. 安全说明
- 穿透式认证流程（AppID + AuthCode）
- 数据本地存储，不外传
- 交易密钥安全管理
